Welcome to Tap Away Block Puzzle 3D. This guide covers the most important editor workflows: creating levels, applying skins, using the in-game shop, and tuning power-ups.

Quick Setup

  1. Open the project from the Unity Asset Store and go to the Game Demo Scene (Assets/Fly Studios Games/Tap Away Block Puzzle 3D/Scenes/Game Demo Scene).
  2. Locate the LevelManager GameObject in the scene and assign required references:
    • SingleBlockPrefab ‚Äî prefab for a single block.
    • LevelContainer ‚Äî parent transform for instantiated blocks.
    • UIManager, ShopManager, CameraControler ‚Äî optional but recommended.
  3. Run the scene to verify the demo levels appear and UI updates correctly.

Tip: Level and shop data are ScriptableObjects. Changing assets in the Project window updates behavior without recompiling scripts.

üß± Level Creation (LevelData)

Levels are defined using LevelData ScriptableObjects. Each LevelData contains block positions, directions and a simple generation interface.

Creating a LevelData asset

  1. In Project window: Create ‚Üí TapAway ‚Üí LevelData (or use the project's Create menu).
  2. Select the asset and in the inspector adjust Grid Length, Grid Height, and Seed to generate variations.
  3. Press Generate (if available in custom editor) to populate the internal block list.

Generated levels are designed to be solvable (forward release algorithm). You can also manually edit block data in the Level Editor window.

Using Level Editor

The included Level Editor window lets you open a dedicated editor scene, place blocks, edit directions and save to LevelData:

  • Open Tools ‚Üí Level Editor.
  • Assign a Block Prefab and a LevelData asset.
  • Use the editor scene preview to move/rotate blocks and sync changes back to the asset.

üé® Skins & Shop

Skins are ScriptableObjects (ShopSkinData) and the shop UI is built from a prefab element (ShopSkinElement).

Creating a new skin

  1. Create asset: Create ‚Üí Tap Away Block Puzzle 3D ‚Üí New Skin.
  2. Fill fields: id (unique), displayName, price, icon, material, arrowColor.
  3. Open ShopManager in scene and add the new skin to the skins list. Assign shopSkinElementPrefab and contentRoot.

Buying/selection is handled by ShopManager. Selected skin is applied to all active blocks immediately.

Notes on persistence

Ownership and selected skin are stored in PlayerPrefs. Keys used by the system are visible in the ShopManager implementation (e.g. SkinOwned_{id}, SelectedSkin).

‚ö° Power-ups & Tutorial

Smash Power-up

Smash is a single-use power-up allowing removal of a block. Purchase via shop, then use Use Smash in UI to select a block to remove.

  • Buy: LevelManager.BuySmash() deducts coins and increases inventory.
  • Use: LevelManager.StartRemoveMode() activates selection; LevelManager.ConfirmRemoveAtBlock(block) consumes the item.

Tutorial steps

The tutorial system uses TutorialManager with ordered step GameObjects. It listens to block activations (Block.OnBlockActivated) and camera rotation to advance steps.

üîå Integration & Common Hooks

Key scripts and hooks

  • LevelManager ‚Äî main game flow, coins, progress, power-ups.
  • UIManager ‚Äî all UI references, progress slider, shop panels.
  • Block ‚Äî per-block behavior: movement, Smash(), ApplySkin().
  • ShopManager / ShopSkinData / ShopSkinElement ‚Äî skin purchasing and selection.
  • CameraControler ‚Äî framing and rotation events used by tutorial.

Adding new content

  1. Create LevelData or Skin assets via Create menu.
  2. Assign assets in scene managers (LevelManager, ShopManager).
  3. Test in play mode. Use the Level Editor for quick iteration.

üõ† Editor Tips & Troubleshooting

Common issues

  • Missing prefab references ‚Äî assign SingleBlockPrefab to LevelManager.
  • UI fields null ‚Äî open UIManager and connect Text, Slider and Buttons in inspector.
  • Shop not responding ‚Äî ensure ShopManager.levelManager is assigned.

Level Editor workflow

Use the Level Editor window to preview and sync changes. When syncing, the editor populates LevelData.blocks from scene children positions. Use the grid unit size to match runtime spacing.

‚ùì FAQ

Q: How do I create a custom level size?
A: Open a LevelData asset and set Grid Length and Grid Height (2-10). Use the Level Editor to fine-tune block positions.

Q: How to add more power-ups?
A: Extend LevelManager with new inventory keys and UI in UIManager; implement appropriate runtime effects in LevelManager or a dedicated PowerUpManager.
